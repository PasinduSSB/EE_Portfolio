Detailed Project

Microgrid Energy Management System

A rule-based EMS project for coordinating solar PV generation, battery energy storage, load demand, and grid exchange across a 24-hour operating profile. The system supports dispatch analysis, battery SOC tracking, renewable self-consumption review, and cost-oriented microgrid operation.

Back to Home

Problem Statement

Microgrids with high renewable penetration face continuous mismatch between PV generation and electrical load demand. Excess solar power may be wasted when the battery is full, while demand peaks can increase grid imports when the battery is not dispatched effectively. The EMS must decide when to use PV, charge storage, discharge storage, import from the grid, or export surplus energy.

Methodology

The model evaluates each hourly time step using load, PV output, battery SOC, battery power limits, SOC boundaries, and grid exchange conditions. PV power is prioritized for local load supply. Surplus PV charges the battery, while remaining surplus is exported or curtailed. During deficits, the battery discharges within its limits before importing power from the grid.

Tools Used

  • Python for EMS logic, dispatch calculations, and cost analysis
  • Streamlit for the interactive dashboard interface
  • Pandas for time-series data handling and result tables
  • NumPy for numerical operations and energy balance calculations
  • Matplotlib or Plotly for engineering graphs and dashboard plots

Results

The EMS produces hourly dispatch outputs including PV used by the load, battery charge and discharge power, SOC variation, grid import, grid export, and estimated operating cost. The results help evaluate renewable utilization, battery sizing, grid dependency, and the effect of dispatch rules on microgrid performance.
